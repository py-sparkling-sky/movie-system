# 观影系统架构设计文档

## 1. 系统概述

观影系统是一个基于Web的电影浏览和管理平台，集成了TMDB（The Movie Database）API，为用户提供电影浏览、搜索、收藏、观影记录等功能。

## 2. 技术架构

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                      前端层                              │
│  HTML5 + CSS3 + JavaScript + Bootstrap 5               │
└─────────────────────────────────────────────────────────┘
                          ↓ HTTP/HTTPS
┌─────────────────────────────────────────────────────────┐
│                      后端层                              │
│  Node.js + Express.js 5.2.1                             │
│  ├── Routes (路由层)                                     │
│  ├── Services (业务逻辑层)                               │
│  ├── Models (数据模型层)                                 │
│  ├── Middlewares (中间件)                               │
│  └── Utils (工具层)                                     │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                   数据存储层                             │
│  ├── MySQL 8.0 (关系型数据库)                           │
│  ├── TMDB API (电影数据源)                              │
│  └── Cache (内存缓存)                                   │
└─────────────────────────────────────────────────────────┘
```

### 2.2 技术栈详情

#### 前端技术栈
- **HTML5 + CSS3**: 页面结构和样式
- **JavaScript (ES6+)**: 业务逻辑
- **Bootstrap 5.3.0**: UI框架
- **Bootstrap Icons**: 图标库
- **Fetch API**: HTTP请求
- **LocalStorage**: 本地存储

#### 后端技术栈
- **Node.js 18+**: 运行环境
- **Express.js 5.2.1**: Web框架
- **Sequelize 6.37.8**: ORM框架
- **MySQL 8.0**: 数据库
- **JWT**: 身份认证
- **Bcrypt**: 密码加密

#### 第三方服务
- **TMDB API**: 电影数据源
- **阿里云RDS**: 生产环境数据库

## 3. 系统模块设计

### 3.1 用户管理模块

**功能**:
- 用户注册、登录
- 用户信息管理
- JWT Token认证
- 权限控制（普通用户/管理员）

**关键文件**:
- `backend/src/models/User.js`: 用户模型
- `backend/src/services/userService.js`: 用户业务逻辑
- `backend/src/routes/userRoutes.js`: 用户路由
- `backend/src/middlewares/auth.js`: 认证中间件

### 3.2 电影浏览模块

**功能**:
- 电影列表展示（分页）
- 电影详情查看
- 电影搜索
- 分类筛选
- 热门电影推荐

**数据来源策略**:
1. 优先从TMDB API获取数据
2. TMDB不可用时降级到本地数据库
3. 使用缓存减少API调用

**关键文件**:
- `backend/src/services/tmdbService.js`: TMDB服务
- `backend/src/services/movieService.js`: 电影业务逻辑
- `backend/src/utils/cache.js`: 缓存管理
- `backend/src/utils/rateLimiter.js`: API限流

### 3.3 观影管理模块

**功能**:
- 电影收藏
- 观影记录
- 观看进度管理

**关键文件**:
- `backend/src/models/Favorite.js`: 收藏模型
- `backend/src/models/WatchRecord.js`: 观影记录模型
- `backend/src/services/watchService.js`: 观影业务逻辑

### 3.4 后台管理模块

**功能**:
- 数据统计
- 电影管理（添加、编辑、删除）
- 用户管理

**关键文件**:
- `backend/src/routes/adminRoutes.js`: 管理员路由
- `backend/src/services/adminService.js`: 管理业务逻辑

### 4.2 数据表说明

#### users表
- 存储用户基本信息
- 支持用户名和邮箱唯一性约束
- 密码使用Bcrypt加密
- 支持角色区分（普通用户/管理员）

#### movies表
- 存储电影基本信息
- 支持评分、观看次数统计
- 支持状态管理（上架/下架）

#### genres表
- 存储电影分类信息
- 与电影多对多关系

#### favorites表
- 存储用户收藏关系
- 用户-电影唯一性约束

#### watch_records表
- 存储观影记录
- 记录观看进度和时长

## 5. API设计

### 5.1 RESTful API规范

所有API遵循RESTful设计规范：
- 使用HTTP方法表示操作（GET、POST、PUT、DELETE）
- 使用URL表示资源
- 使用HTTP状态码表示结果
- 统一的响应格式

### 5.2 响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 5.3 认证机制

- 使用JWT Token认证
- Token放在请求头Authorization字段
- 格式：`Bearer <token>`
- Token有效期：7天

## 6. TMDB集成设计

### 6.1 集成策略

```
┌──────────────┐
│  前端请求    │
└──────────────┘
       ↓
┌──────────────┐
│ movieService │
└──────────────┘
       ↓
   检查配置
       ↓
┌──────┴──────┐
│             │
↓             ↓
TMDB可用    TMDB不可用
│             │
↓             ↓
tmdbService  本地数据库
│             │
└──────┬──────┘
       ↓
┌──────────────┐
│  返回数据    │
└──────────────┘
```

### 6.2 缓存机制

- 使用内存缓存减少API调用
- 默认缓存时间：1小时
- 支持自定义TTL
- 定期清理过期缓存

### 6.3 限流机制

- 控制API调用频率
- 默认每分钟最多40次请求
- 请求间隔：250ms
- 避免超过TMDB限流限制
